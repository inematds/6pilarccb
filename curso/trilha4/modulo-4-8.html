🗓️ Mapeamento do Workflow Diario

O mapeamento do workflow diario define qual prompt usar em cada fase do seu dia de trabalho. Em vez de escolher prompts aleatoriamente, voce segue um fluxo previsivel que maximiza produtividade e qualidade.

🎯 Workflow Diario Ideal

08:00 - Inicio do Dia
Abra o Claude Code → 95% Confidence → Estude o que vai trabalhar hoje
08:30 - Planejamento
Plan Mode Blueprint → Gere o blueprint da tarefa principal do dia
09:00-12:00 - Construcao
Feature Blueprint + Build With Tests → Implemente passo a passo
12:00 - Meio do Dia
/compact → Compacte o contexto se a sessao esta longa
14:00-17:00 - Continuacao + Debug
Structured Bug Report quando encontrar bugs → Continue building
17:00 - Final do Dia
Brutal Audit → Revise tudo antes do commit final

💡 Dica Pratica

Adapte os horarios ao seu ritmo. O importante nao e a hora, e a sequencia: Estudar → Planejar → Construir → Debugar → Revisar. Essa progressao garante que voce sempre comeca informado e termina com qualidade.

🎯 95% Confidence para Tarefas Complexas

O 95% Confidence e obrigatorio para tarefas complexas: mudancas em multiplos arquivos, alteracoes em areas criticas (auth, pagamentos, dados), ou entrada em projetos desconhecidos. Pular esse passo em tarefas complexas e a causa #1 de retrabalho.

🎯 Quando Usar (e Quando Nao)

USAR 95% Confidence

  • ✓Tarefa toca 3+ arquivos
  • ✓Area critica (auth, pagamento, dados)
  • ✓Projeto novo ou desconhecido
  • ✓Mudanca arquitetural
  • ✓Bug complexo/intermitente

NAO precisa de 95%

  • ✗Renomear variavel
  • ✗Ajustar texto/CSS
  • ✗Adicionar log/comentario
  • ✗Bug obvio em 1 arquivo
  • ✗Tarefa repetitiva ja conhecida

💡 Dica Pratica

Na duvida, use. O custo de 5 minutos de estudo e insignificante comparado com o custo de 2 horas de retrabalho por ter comecado sem entender o contexto.

📋 Plan Mode para Building

O Plan Mode e seu melhor amigo durante a fase de construcao. Ele garante que voce tem um mapa antes de comecar a navegar. Sem plano, voce constroi no escuro e descobre problemas tarde demais.

🎯 Estrategia de Plan Mode 

FLUXO: Plan Mode → Approve → Implement Step 1 → Review → Implement Step 2 → ...

Comandos chave durante building:
1. "Use ultrathink para planejar isso" → Ativa planejamento profundo
2. "Implemente apenas o passo 1" → Execucao incremental
3. "PARE. Me mostre o que fez antes de continuar" → Gate de controle
4. "Isso mudou o comportamento? Nao deveria." → Guardrail
5. "Rode os testes antes de prosseguir" → Verificacao continua

Nunca diga: "Faz tudo de uma vez"
Sempre diga: "Faz o proximo passo e para"

💡 Dica Pratica

Se o plano tem mais de 5 passos, decomponha em sub-planos. Um plano com 15 passos e tao util quanto nenhum plano. Mantenha cada plano entre 3-5 passos actionaveis.

4

🔍 Brutal Audit para Encerramento

O Brutal Audit e o ultimo prompt antes de encerrar uma sessao ou fazer um commit importante. Ele e seu gate de qualidade final - nada vai para producao sem passar por ele.

🎯 Ritual de Encerramento 

Antes de encerrar esta sessao, faca:

1. Brutal Audit de tudo que foi construido/modificado hoje
2. Liste todos os arquivos modificados (git diff --name-only)
3. Para cada arquivo: ha algo que me preocupa?
4. Testes: todos passam? Ha testes novos necessarios?
5. Documentacao: CLAUDE.md precisa ser atualizado?
6. TODO list: o que ficou pendente para a proxima sessao?

Classifique o status geral:
- PRONTO PARA COMMIT: Tudo limpo, pode ir
- PRECISA AJUSTE: Pequenas correcoes antes de commitar
- NAO COMMITAVEL: Problemas serios, nao commite assim

💡 Dica Pratica

O item "TODO list para proxima sessao" e crucial. Ao iniciar a sessao seguinte, voce sabe exatamente onde parou e o que precisa fazer. Salve esse TODO no CLAUDE.md ou em um arquivo de sessao.

📚 Teach Me para Aprendizado Continuo

O Teach Me nao e apenas para momentos de duvida - e uma estrategia de aprendizado continuo. Use-o sistematicamente para aprofundar seu conhecimento enquanto trabalha, transformando cada sessao em uma oportunidade de crescimento.

🎯 Momentos Ideais para Teach Me

  • • O Claude usou um padrao que voce nao conhece - "Me ensine o padrao [X] que voce acabou de aplicar"
  • • Uma decisao arquitetural nao e obvia - "Por que essa abordagem e melhor que [alternativa]?"
  • • Um bug revelou um conceito que voce nao domina - "Me ensine [conceito] para que eu evite esse tipo de bug"
  • • Final da sessao com tempo livre - "Me ensine algo interessante sobre o codigo que trabalhamos hoje"

💡 Dica Pratica

Mantenha um "diario de aprendizado" no projeto. Cada vez que o Teach Me revelar algo valioso, anote brevemente no CLAUDE.md ou em um arquivo separado. Em 3 meses, voce tera um acervo de conhecimento contextualizado.

🧹 Estrategia de Limpeza de Contexto

A limpeza de contexto e tao importante quanto os prompts em si. Uma sessao com contexto poluido produz resultados piores. Saber quando compactar, quando iniciar sessao nova e quando limpar e essencial para manter a qualidade.

🎯 Regras de Limpeza

Regra 1: /compact a 70% de contexto
Quando sentir que as respostas estao ficando genericas ou o Claude comecar a esquecer instrucoes, compacte. Use /cost para monitorar o uso de contexto.
Regra 2: Nova sessao para nova tarefa
Se voce vai mudar completamente de tarefa (ex: de building para debugging em outro modulo), inicie uma sessao nova. Contexto residual de outra tarefa confunde.
Regra 3: /clear para foco visual
Use /clear quando a tela esta poluida mas voce quer manter o contexto. Limpa a tela, mantem a memoria.
Regra 4: Compact com instrucao customizada
Use /compact "Mantenha foco em [tarefa X] e descarte discussoes sobre [Y]" para direcionar o que preservar.
