MODULO 5.9

🔄 Sincronizando Skills entre Code e Cowork

Aprenda as 3 formas de manter suas skills sincronizadas entre Claude Code (terminal) e Cowork (desktop).

1

❓ O Problema da Sincronizacao

Skills criadas no Claude Code ficam em .claude/commands/ como arquivos markdown. No Cowork, skills aparecem como slash commands nativos. Esses dois mundos nao sincronizam automaticamente.

💡 Conceito Principal

  • •Claude Code le skills de .claude/commands/
  • •Cowork tem skills built-in + plugins instalados
  • •Existem 3 opcoes para sincronizar entre os dois
2

📁 Opcao 1: Selecionar Pasta de Skills

A forma mais rapida: no Cowork, selecione a pasta onde suas skills do Claude Code estao. O Claude pode le-las e segui-las, mas nao aparecem como slash commands.

💡 Conceito Principal

  • •No Cowork, selecione ~/.claude/commands/
  • •Claude pode ler e seguir as instrucoes dos arquivos .md
  • •Rapido mas sem slash commands nativos

⚡ Dica Pratica

Use esta opcao quando precisa de acesso rapido sem configuracao. Ideal para testar skills antes de empacotar.

3

📦 Opcao 2: Empacotar como Plugin

A melhor solucao a longo prazo: use /skill-creator ou /create-cowork-plugin no Cowork para transformar suas skills em um arquivo .plugin.

💡 Conceito Principal

  • •Plugins empacotam skills + MCPs + tools em um arquivo
  • •Instale uma vez, use em todos os projetos
  • •Skills aparecem como slash commands nativos
4

🔨 Opcao 3: Recriar no Cowork

Para skills simples, peca ao Claude no Cowork para criar a skill. Leva segundos e fica formatada corretamente para o ambiente Cowork.

💡 Conceito Principal

  • •Ideal para skills curtas e simples
  • •Descreva o que a skill faz e o Claude cria automaticamente
  • •Formatacao nativa do Cowork garantida
5

🛠 O Skill Creator

O /skill-creator e a ferramenta oficial para criar, testar e empacotar skills de forma profissional.

💡 Conceito Principal

  • •Cria skills com template padronizado
  • •Roda evals para testar a qualidade
  • •Otimiza a descricao para melhor triggering
6

🎯 Melhores Praticas

Regras para manter suas skills organizadas e sincronizadas entre ambientes.

💡 Conceito Principal

  • •Mantenha um repositorio central de skills
  • •Versione suas skills como qualquer outro codigo
  • •Use plugins para skills que voce usa em multiplos projetos
  • •Mantenha skills especificas do projeto em .claude/commands/
